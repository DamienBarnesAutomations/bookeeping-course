# Implementation Plan — QQI Level 5 Course Website

A fully static Single Page Application (SPA) hosted on GitHub Pages, built with plain HTML, CSS, and JavaScript. No build tools, no frameworks, no backend.

---

## Architecture

The site fetches Markdown files from the repo at runtime using the browser `fetch()` API and renders them to HTML using `marked.js` (CDN). A `manifest.json` file (generated by a Python script) tells the SPA what files exist and how they are organised.

```
bookeeping-course/
├── index.html            ← SPA shell
├── styles.css            ← Design system (dark/light themes, responsive)
├── app.js                ← Client router, markdown renderer, worksheet engine
├── manifest.json         ← Auto-generated course file index (45 files, 9 categories)
├── generate_manifest.py  ← Re-run this whenever .md files are added/renamed
├── bookkeeping/          ← 5N1354 — 10 lessons + deep dive
├── payroll/              ← 5N1546 — 10 lessons + 2x deep dive
├── exercises/            ← Bookkeeping & payroll worksheets
├── exercise-solutions/   ← Worked solutions
├── templates/            ← 10 blank practice forms (interactive in the SPA)
├── mock-projects/        ← Full QQI mock assessments
├── software-guides/      ← Sage/QuickBooks and Thesaurus/Sage Payroll guides
├── reference/            ← Glossary (70+ terms) + cheat sheets
└── assessment/           ← Assessment prep
```

## Files

### `generate_manifest.py`
- Walks all 9 course directories.
- Reads the first `# Heading` from each `.md` file to use as the display title.
- Writes `manifest.json` with file paths, titles, category metadata, and icon names.
- Run locally whenever course files change: `python generate_manifest.py`

### `index.html`
Semantic HTML5 shell with:
- `<header>` — brand logo (home link), progress bar, theme toggle, "Mark Complete" button.
- `<aside>` — sidebar with grouped navigation links (desktop: always visible; mobile: slide-in drawer).
- `<main>` — two views: **Dashboard** and **Reader**, toggled by the router.
- `<nav>` — mobile bottom bar (Home / Bookkeeping / Payroll / Templates).
- CDN deps: `marked.js`, `lucide` icons, Google Fonts (Outfit + Inter).

### `styles.css`
- **CSS custom properties** for all colours, radii, and transitions — supports instant dark/light switching.
- **Dark theme** (default): `#0b0f19` background, indigo/cyan accent gradient.
- **Light theme**: `#f8fafc` background, same accent palette at lower saturation.
- **Glassmorphism** header and mobile nav (`backdrop-filter: blur`).
- **Desktop**: sidebar always visible at 320px, main content offset by `margin-left`.
- **Mobile** (`≤768px`): sidebar hidden behind drawer, main content full-width with `padding-bottom: 80px` to clear the fixed bottom nav.

### `app.js`

#### Hash Router
- Listens to `hashchange` — routes `#` to the Dashboard, `#category/file.md` to the Reader.
- Updates sidebar active state and mobile bottom nav active tab on every route change.

#### Manifest & Progress Loader
- Fetches `manifest.json` on init.
- Loads `completed_lessons` array from `localStorage`.
- Drives the global progress bar, dashboard stats (overall %, bookkeeping %, payroll %).

#### Sidebar Renderer
- Builds grouped `<ul>` lists from manifest categories.
- Each link shows a `circle` or `check-circle` Lucide icon based on completion state.

#### Reader / Markdown Renderer
- Fetches the `.md` file, parses it with `marked.parse()`, injects HTML into `.prose`.
- Sets breadcrumbs, Mark Complete button state, Next/Previous pagination.
- Builds floating Table of Contents from `h2`/`h3` headings (desktop only, >1024px).

#### Interactive Worksheet Engine (Templates only)
- Detects `templates/` path — shows the "Interactive practice" toggle.
- In practice mode, replaces empty table cells with `<input>` fields (text or numeric, inferred from column header keywords).
- Totals rows become auto-calculated display cells.
- **Sales/Purchases Daybooks**: entering Net auto-fills VAT @23% and Gross (bidirectional).
- **VAT Return Worksheet**: full T1–T6 box calculation — Output VAT, Input VAT, and Payable/Refundable balance all auto-compute.
- Reset button re-renders the lesson to clear all inputs.

#### Glossary Page
- Custom UI instead of raw markdown render.
- Parses `| **Term** | Definition |` rows with a regex.
- Tags each term as Bookkeeping or Payroll based on its position in the source file.
- Live search filters by word or definition text on every keystroke.

## Deploying to GitHub Pages

1. Commit the new files:
   ```bash
   git add index.html styles.css app.js manifest.json generate_manifest.py
   git commit -m "feat: add course website SPA"
   git push
   ```

2. In GitHub: **Settings → Pages → Source → main branch, / (root) → Save**

3. Site will be live at:
   ```
   https://DamienBarnesAutomations.github.io/bookeeping-course/
   ```

> If you add or rename `.md` files, re-run `python generate_manifest.py` and commit the updated `manifest.json`.
